RouteTrack Push Send TypeScript Fix

Problem:
Vercel build failed in:
src/app/api/push/send/route.ts

Reason:
The manually declared Firestore callback type used a custom ref.set() signature.
It did not exactly match Firebase Admin Firestore's overloaded DocumentReference.set() type.

Fix:
The route now uses a small structural PushDeviceSnapshot type and casts
the Firestore result once before filtering. This removes the overload conflict
without changing notification behavior.

Replace:
src/app/api/push/send/route.ts

Steps:
1. Extract this ZIP.
2. Copy the included src folder into the existing RouteTrack project.
3. Choose Replace the file in the destination.
4. Run:
   npm run build
5. Push:
   git add .
   git commit -m "Fix push notification Firestore TypeScript overload"
   git push origin main
6. Wait for the existing Vercel deployment to become Ready.

No npm install is required.
No Firestore Rules update is required.
No environment-variable change is required.
